Firebase Cloud Setup

📡 One-Time Setup
Connect to Firebase (free) for online accounts & cloud database.
1. Visit console.firebase.google.com → Create project
2. Build → Authentication → Sign-in method → Email/Password → Enable
3. Build → Firestore Database → Create → start in production mode
4. ⚙ Project Settings → Your apps → Web (</>) → Register → copy values below
5. Firestore → Rules tab → paste rules below → Publish
📋 Click for Firestore Security Rules
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { function si() { return request.auth != null; } function rl() { return get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role; } function isAdmin() { return si() && rl() == 'admin'; } function isDoctor() { return si() && rl() == 'doctor'; } function isStaff() { return si() && rl() in ['admin','doctor','receptionist']; } match /users/{uid} { allow read: if si(); allow create: if request.auth.uid == uid; allow update: if isAdmin() || request.auth.uid == uid; allow delete: if isAdmin(); } match /patients/{id} { allow read: if isStaff(); allow create, update: if isStaff(); allow delete: if isAdmin(); } match /examinations/{id} { allow read: if isStaff(); allow create, update, delete: if isAdmin() || isDoctor(); } match /sales/{id} { allow read, write: if isStaff(); } match /repairs/{id} { allow read, write: if isStaff(); } match /expenditures/{id} { allow read, write: if isStaff(); } match /frames/{id} { allow read: if isStaff(); allow write: if isAdmin() || rl()=='receptionist'; } match /lenses/{id} { allow read: if isStaff(); allow write: if isAdmin() || rl()=='receptionist'; } match /accessories/{id} { allow read: if isStaff(); allow write: if isAdmin() || rl()=='receptionist'; } } }

Multi-Device & Deployment Guide
✓ Cloud-synced — Once configured, your data lives in Firebase. Same login from any device shows the same data.

🚀 Hosting Options

Option A — Replit (easiest, recommended)
1. Go to replit.com → Sign up free
2. Click Create Repl → choose template "HTML, CSS, JS"
3. Delete the existing index.html → upload this file and rename it to index.html
4. Click Run at top → Replit gives you a URL like your-repl.your-name.repl.co
5. Click the URL → use it on any phone, laptop, Chrome account
6. Each staff member opens that URL → enters your Firebase config once → signs in
Option B — Netlify Drop (drag & drop)
Go to app.netlify.com/drop → drag this HTML file → instant URL → share with your staff.
Option C — Firebase Hosting (most professional)
Install: npm install -g firebase-tools → firebase login → put HTML in folder → firebase init hosting → firebase deploy. URL: your-project.web.app
Option D — Email/share the file directly
Send the HTML file via WhatsApp/email. Each device opens it once, enters Firebase config (saved per-device), then signs in normally.

📋 What each user needs

  • The URL (or HTML file) — same for everyone
  • Same Firebase config (only entered once per device — saves automatically)
  • Their own email + password
  • Internet connection

🔐 Roles & how to assign them

  • First user to register → automatic Admin
  • Sign up with code DOCTOR2025 → Doctor (clinical access)
  • Sign up with code ADMIN2025 → Admin
  • Sign up with no code → Receptionist (sales, expenditure, view-only patient details)
  • Admin can change anyone's role in the Users page

🎯 What each role sees

  • Admin: everything — dashboard, patients, sales, stock, repairs, expenditure, reports/P&L, user management
  • Doctor: dashboard, patients (full clinical access — case history, refraction, anterior/posterior segment, diagnosis, treatment, management)
  • Receptionist: dashboard, patient details (name/age/sex/address + diagnosis & prescription view-only — cannot add/edit clinical history), sales, sales history, prescriptions, expenditure
⚠ Important: Don't forget Firestore Security Rules (Step 5). Without them, data is publicly accessible.
